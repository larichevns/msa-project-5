# Сравнительный анализ технологий для пакетной обработки данных

## Задача
Ежедневная генерация кастомных CSV прайс-листов для B2B-клиентов в 6:00 утра на основе данных PostgreSQL.

## Параметры системы
- **Объем данных**: 5 000 – 10 000 строк результата
- **Исходные таблицы**:
  - products: ~5 000–10 000 строк
  - categories: ~50–200 строк
  - clients: ~100–500 строк
  - client_prices: ~10 000–20 000 строк
- **Инфраструктура**: микросервисы в Kubernetes

## Выбранное решение: Kubernetes CronJob

### Обоснование выбора

#### 1. **Идеальное соответствие задаче**
- Задача представляет собой простую ETL операцию без сложной обработки
- Не требуется управление сложными зависимостями между задачами
- Выгрузка данных - это единичная операция с JOIN таблиц и записью в CSV

#### 2. **Нативная интеграция с существующей инфраструктурой**
- Система уже работает в Kubernetes
- Не требует развертывания дополнительных компонентов
- Использует существующие ресурсы кластера

#### 3. **Минимальная ресурсоемкость**
- Контейнер с Python/Node.js скриптом потребляет ~50-100MB памяти
- Запускается только в момент выполнения (6:00)
- Не занимает ресурсы в остальное время

#### 4. **Простота реализации и поддержки**
- Простой SQL запрос с JOIN таблиц
- ~100 строк кода на Python/Node.js
- Легко понимается и поддерживается любым разработчиком

#### 5. **Встроенные возможности K8s**
- Автоматические retry при сбоях
- Логирование через kubectl logs
- Мониторинг через Prometheus metrics
- Алерты через существующую систему мониторинга

#### 6. **Масштабируемость**
- При росте данных до 100K+ строк - увеличение resources limits
- При необходимости параллельной обработки - Job с parallelism
- Возможность миграции на Spark/Airflow в будущем без изменения архитектуры

### Почему не другие решения?

**Spring Batch**: Избыточен для простой задачи, требует JVM и Spring инфраструктуры

**Apache Airflow**: Оверкилл для единичной задачи без зависимостей, требует отдельного кластера

**Apache Spark**: Предназначен для обработки больших данных (TB+), избыточен для 10K строк

## Вывод

Kubernetes CronJob - оптимальное решение для данной задачи, обеспечивающее:
- ✅ Минимальную сложность
- ✅ Максимальную интеграцию с существующей инфраструктурой
- ✅ Низкое потребление ресурсов
- ✅ Простоту поддержки и мониторинга
- ✅ Возможность легкой миграции на более сложные решения при необходимости
